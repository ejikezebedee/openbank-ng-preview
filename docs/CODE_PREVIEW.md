# OpenBank NG Code Preview

## Full Source Status

The full source-code repository is private.

This public preview does not expose the complete implementation. It gives buyers enough context to evaluate the product category, package structure, commercial boundary, and access process.

## Full Package Structure

```text
apps/customer-web/      Customer banking interface
apps/admin-web/         Admin operations dashboard
services/api/           Backend API and banking services
packages/shared/        Shared types, constants, and validation helpers
db/                     PostgreSQL schema notes and migration
docs/                   Setup, API, deployment, handoff, and release docs
sales/                  Sales, FAQ, launch, and support materials
release/                Release audit, checklist, manifest, access SOP
```

## Buyer Code Examination Options

Approved buyers may examine the full code through one of these methods:

- Private GitHub read-only access.
- Screen-share review.
- Validated private ZIP/hash after purchase.
- Custom diligence review under a separate agreement.

## What Buyers Should Examine

- API route structure.
- Ledger and transaction boundaries.
- KYC and risk workflows.
- Admin operations model.
- Shared Nigerian banking primitives.
- PostgreSQL schema and repository boundaries.
- Setup, deployment, security, and release documentation.

## Access Gate

Full code access requires buyer approval, accepted license terms, payment confirmation, approved support scope, confirmed GitHub invite target, and seller approval.
